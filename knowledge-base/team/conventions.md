# Team Conventions

## Git Conventions
- Branch naming: `feature/ENG-1234-short-description`
- Commit messages: conventional commits (`feat:`, `fix:`, `docs:`, `chore:`)
- PRs: require at least 1 approval before merge
- Squash merge to main

## Code Style
- Follow language-specific linters (configured per repo)
- Write tests for all new features
- Document public APIs

## Communication
- Use Linear for task tracking
- Use Slack for async communication
- Use Google Meet for synchronous discussions
- Post deploy announcements in #deployments

## Documentation
- All architecture decisions recorded as ADRs
- Runbooks for any operational procedure
- README.md required for every repo
